Setup Guide · ST Skills Pack v1.5.4

How do I
get started?

Follow these steps to install Cursor, configure the ST Skills Pack, and set up all required MCP servers and extensions. Estimated setup time: ~20 minutes.

01

Install Cursor & sign in

Download and install Cursor from cursor.com. On first launch, sign in using your Sitetracker Google account (@sitetracker.com).

02

Clone st-cursor-skills from Bitbucket

This repo contains the ST Skills Pack — rules, hooks, commands, and skills tailored for Sitetracker development.

git clone git@bitbucket.org:sitetraker/st-cursor-skills.git
💡

Optional: Before copying files, open mcp.json inside the repo and review the MCP server configuration. Decide which servers you want to bring into your ~/.cursor folder — you can always add or remove them later.

03

Clone strk-mcp from Bitbucket

This MCP server lets Cursor LLM browse Sitetracker package source code directly — enabling context-aware answers about the managed package without leaving your IDE.

git clone git@bitbucket.org:sitetraker/strk-mcp.git
04

Clone the Sitetracker repo from Bitbucket

This is the main Sitetracker managed package source repo. strk-mcp needs a local clone to index and serve package metadata to Claude.

git clone git@bitbucket.org:sitetraker/sitetracker.git
05

Clone the code-review-graph fork from GitHub

This is the Cursor/VS Code extension that adds graph-aware code review capabilities — it understands your Salesforce org's dependency graph when reviewing changes.

git clone https://github.com/renan-brasilio/code-review-graph.git

You'll build and install this extension locally in step 8.

06

Back up your existing Cursor config

Optional

If you have existing rules, hooks, commands, or skills in your ~/.cursor folder that you want to keep, back them up before the next step overwrites anything.

# Back up your entire .cursor config cp -r ~/.cursor ~/.cursor-backup
07

Copy files from st-cursor-skills to your .cursor folder

Copy the contents of the repo's .cursor folder into your local ~/.cursor directory.

# Replace the path with where you cloned the repo cp -r ~/path/to/st-cursor-skills/.cursor/* ~/.cursor/

This installs the ST rules, hooks, commands, and skills that were shown during the presentation.

08

Install code-review-graph from your local repo

code-review-graph is a CLI tool — install it directly from your local clone using uv.

# Go into the cloned repo and install with uv cd code-review-graph uv tool install .

Once installed, the code-review-graph command will be available in your terminal.

⚠️

Beta warning: avoid running pipx upgrade-all or uv tool upgrade --all during this beta period — these commands may replace your local fork install with the original upstream version.

09

Open Cursor and verify all MCPs are configured

Open Cursor, then go to Settings → MCP Servers. All configured servers should show a green connected status.

If any server shows red or disconnected, check:

· The repo path in mcp.json points to where you actually cloned it
· Any required environment variables (e.g. STRK_REPO_ROOT) are set
· Node.js / the server's runtime is installed and accessible

10

Build the graph in your target repo

Navigate to the Salesforce repo you want to work in and run the build command to generate the dependency graph.

# Run from the root of your target Salesforce repo cd /path/to/your/salesforce-repo code-review-graph build

This indexes the repo's metadata and builds the graph that powers context-aware code review. Run it again whenever you pull significant changes.

11

Vibe code 🎉

You're all set. Open any Salesforce repo in Cursor and start building with your new AI superpowers — context-aware code review, graph-aware impact analysis, and a full suite of ST slash commands at your fingertips.
